# 순열과 조합

## 경우의수

어떤 조건을 만족하는 **서로 다른 선택의 개수**

```
예시
• 동전 앞/뒤 → 2가지
• 주사위 → 6가지
• 티셔츠 3벌 × 바지 2벌 → 3 × 2 = 6가지
```

## 팩토리얼

모두 줄 세우는 경우의 수

$$
n! = n \times (n-1) \times (n-2) \times \cdots \times 1
$$

### 팩토리얼의 직관적인 생각

4명이 줄을 선다고 가정합니다

처음 줄에 설 수 있는 사람은 모두 가능합니다.
한 사람이 서고 나면,
그다음 자리에는 남은 사람만 설 수 있습니다.

- 1번째 사람은 4가지 위치에 줄을 설수있다.
- 2번쨰 사람은 3가지 위치에 줄을 설수있다
- 3번쨰 사람은 2가지 위치에 줄을 설수있다
- 4번쨰 사람은 1가지 위치에 줄을 설수있다

모든 것을 빠짐없이, 순서를 정해서 나열하는 경우의 수를 구할때 팩토리얼을 사용합니다.

## 순열 (Permutation)

서로 다른 n개 중에서 r개를 뽑아 순서를 고려하여 나열하는 것

$$
{}_nP_r = n \times (n-1) \times \cdots \times (n-r+1) = \frac{n!}{(n-r)!}
$$

### 순열에 대한 직관적인 생각

순열을 이해할때 중요한것은 자리입니다.
팩토리얼은 모든것을 순서를 정해 나열하지만, 순열은 r개 만큼의 순서를 고려해 나열합니다.

${}_nP_r$ 도 마찬가지입니다

n개중 r개만 정렬하는것이기 떄문에, 팩토리얼과 다르게 n - r + 1개 만큼 자리가 있습니다

n개중 n개까지 뽑는다면 팩토리얼과 동일하고,
n개중 r개까지 뽑는다면 팩토리얼의 일부까지만 센다고 보면됩니다.

### 예시

3명 A, B, C 중 2명을 줄 세우기
• AB, BA, AC, CA, BC, CB

## 조합 (Combination)

서로 다른 $n$개 중에서
$r$개를 뽑되 순서는 고려하지 않음

$$
{}_nC_r = \frac{n!}{r!(n-r)!}
$$

3명 A, B, C 중 2명을 팀으로 선택
• {A, B}, {A, C}, {B, C}

### 조합에 대한 직관적인 생각

조합은 자리가 존재하지 않습니다.
어떤 자리에 무엇이 왔는지는 중요하지 않고,
누가 포함되었는가만이 중요합니다.
또한 같은 값을 중복해서 선택할 수는 없습니다.

그래서 조합의 결과는
순서가 없는 집합처럼 생각할 수 있습니다.

⸻

예를 들어 A, B, C 세 사람이 있을 때
이 중에서 2명을 선택한다고 해봅시다.

먼저 순서를 고려해서 생각하면 이는 순열이며,
경우의 수는 다음과 같습니다.
• AB
• BA
• AC
• CA
• BC
• CB

총 6가지로, 이는 ${}_3P_2$에 해당합니다.

⸻

하지만 조합의 관점에서 보면
AB와 BA는 서로 다른 선택이 아닙니다.
두 경우 모두 A와 B가 포함되었다는 점에서
완전히 동일한 선택입니다.

즉, 순열로 경우의 수를 세면
같은 조합이 순서만 바뀐 채 여러 번 등장하게 됩니다.

⸻

그렇다면 하나의 조합은
순열에서 몇 번씩 반복해서 등장할까요?

2명을 선택했을 때,
이 두 사람을 나열할 수 있는 방법은 항상 같습니다.
• 첫 번째 자리에 올 수 있는 경우: 2가지
• 두 번째 자리에 올 수 있는 경우: 1가지

즉, 두 사람의 순서를 바꾸는 방법은 항상 $2!$ 가지입니다.

이 성질은 선택된 사람이 무엇이든 동일합니다.
• AB는 2!
• AC도 2!
• BC도 2!

모든 조합은 항상 같은 수(2!)만큼
순열 속에서 중복되어 나타납니다.

⸻

조합 ${}_nC_r$ 은
이처럼 순서만 다른 중복된 경우를 제거하고,
구성만 서로 다른 선택의 개수를 세는 개념입니다.

따라서 순열 ${}_nP_r$ 에 포함된 중복을 없애기 위해
각 조합이 중복되어 등장한 횟수인 $r!$ 만큼을 나누어 줍니다.

이 나눗셈은
경우의 수를 줄이기 위한 임의의 계산이 아니라,
같은 선택을 하나로 묶기 위한 중복 제거 과정입니다.

그래서 결과적으로
• 순열 ${}_nP_r$ 을
• 중복 횟수인 $r!$ 로 나누면
• 중복 없는 조합 ${}_nC_r$ 이 됩니다.

$$
{}_nC_r = \frac{{}_nP_r}{r!}
$$

## 중복순열

이미 선택한것을 다시 선택할수 있을때 순열

$$
n^r
$$

### 예시

n개의 주사위로 r개를 뽑는다고 가정했을때
1개당 6개의 가지수가 있으므로 $6 \times 6 \times .. r개 = n^r$ 입니다

비밀번호도 가지수를 센다면 동일하게 $n^r$ 로 계산할수 있습니다
영문 26자, 숫자 10개로 비밀번호 10자리의 순열의 개수를 구하면 $36^{10}$ 이 됩니다

## 중복조합

종류는 여러 개인데, 같은 것을 여러 번 골라도 될 때 몇 가지 선택이 가능한가?

$$
{}_{n+r-1}C_r
$$

### 예시

A, B, C 세 종류가 있을 때
총 5개를 고르는 문제는 중복조합이다.

이 문제에서 우리는
“어떤 것을 고를 것인가”가 아니라
“각 종류를 몇 개씩 고를 것인가”를 묻는다.

즉,
A 몇 개, B 몇 개, C 몇 개인지의 분배 문제다.

이 분배 문제를 그대로 세기는 어렵기 때문에
수학에서는 이를 조합으로 세기 쉬운 문제로 변환한다.

먼저 선택된 5개를 동일한 기호 O로 생각하고
이를 한 줄로 나열한다.
그리고 A, B, C 세 종류를 구분하기 위해
칸막이 |를 2개(n−1개) 둔다.

이때
O는 실제로 선택한 개수 r개이고,
|는 종류를 구분하기 위한 표현 도구일 뿐
선택 대상이 아니다.

O 5개(r)와 | 2개(n - 1)를 한 줄에 배치하면
각 배치 패턴은
A, B, C 각각의 개수를 하나로 정하게 된다.

따라서 이 문제는
총 n+r−1개의 자리 중에서
O r개의 위치를 정하는 문제로 바뀌며,
이는 조합으로 계산할 수 있다.

그래서 중복조합의 개수는

$$
{}_nH_r =  {}_{n+r-1}C_r
$$

로 표현된다.

## 같은 원소가 있는 순열

바나나 3개, 사과 2개가 있다고 하자
전체 순열을 구한다고 할때 몇가지가 있을까?

생각해보면, 바나나끼리 순열을 구하면 모두 1가지이다.
사과도 1가지로 동일하다

전체 순열인 5! 의 개수중 사과의 중복순열 2!과 바나나의 중복순열 3!은 제외해야한다.

그래서 수식은 아래와 같이 구할수 있다

$$

\frac{5!}{3! \cdot 2!}


$$

이건 ${}_nC_r$ 을 구할떄 중복된 순열을 제거하는것과 동일한 원리이다.

## ${}_7C_2$ 와 ${}_7C_5$ 이 동일한 이유

고른다는 행위는 사실 무엇을 의미할까
“고른다”는 것은 동시에 “안 고른다”이기도 하다

그래서 경우의 수가 같아질 수밖에 없다

# Python Itertools의 순열과 조합

## itertools.permutations

순열을 만드는 함수

```python
import itertools
list(itertools.permutations(['1', '2', '3'], 2))
# [('1', '2'), ('1', '3'), ('2', '1'), ('2', '3'), ('3', '1'), ('3', '2')]
```

## itertools.combinations

조합을 만드는 함수

```python
import itertools
list(itertools.permutations(['1', '2', '3'], 2))
# [('1', '2'), ('1', '3'), ('2', '3')]
```

## itertools.combinations_with_replacement

```python
import itertools
list(itertools.combinations_with_replacement(['1', '2', '3'], 2))
# [('1', '1'), ('1', '2'), ('1', '3'), ('2', '2'), ('2', '3'), ('3', '3')]
```

## 파이썬 deque

```python
from collections import deque


dq = deque()

dq.append(1)        # 오른쪽 추가
dq.append(2)
dq.appendleft(0)    # 왼쪽 추가

print(dq)           # deque([0, 1, 2])

dq.pop()        # 오른쪽 제거
dq.popleft()    # 왼쪽 제거
```

# 그래프와 역방향 그래프에 대한 정리

https://www.acmicpc.net/problem/1238

그래프에서 최단 경로 알고리즘(다익스트라)은
항상 하나의 시작점에서 모든 정점까지의 최단거리만을 계산한다.
즉, 기본적으로는
시작점 → 다른 모든 정점 방향의 거리만 구할 수 있다.

하지만 문제에 따라서는
모든 정점 → 특정 정점으로 가는 최단거리가 필요한 경우가 있다.
이때 각 정점을 시작점으로 다익스트라를 실행하면
시간 복잡도가 크게 증가한다.

이 문제를 해결하기 위해 사용하는 개념이 역방향 그래프이다.

역방향 그래프는
기존 그래프의 모든 간선 방향을 반대로 뒤집은 그래프이다.
예를 들어 A → B (비용 w)라는 간선이 있다면
역방향 그래프에서는 B → A (비용 w)가 된다.

이렇게 그래프를 뒤집으면,
원래 그래프에서의 i → X 최단경로는
역방향 그래프에서의 X → i 최단경로와 동일한 비용을 갖게 된다.
경로의 방향만 바뀌었을 뿐, 지나가는 간선과 비용은 같기 때문이다.

따라서
• 원래 그래프에서 X를 시작점으로 다익스트라를 실행하면
X → i 최단거리를 구할 수 있고
• 역방향 그래프에서 X를 시작점으로 다익스트라를 실행하면
i → X 최단거리를 한 번에 구할 수 있다.

이 방법을 사용하면
다익스트라를 여러 번 실행하지 않고도
모든 정점의 왕복 최단거리 문제를 효율적으로 해결할 수 있다.
