# 다익스트라 문제풀이

## BOJ 1504 — 특정한 최단 경로

- 반드시 지나야 하는 두 정점(v1, v2)이 있을 때는 경유 순서를 분리해서 생각해야 한다.
- 가능한 경로는 `1→v1→v2→N`, `1→v2→v1→N` 두 가지뿐임을 이해했다.
- 최단경로 계산은 6구간처럼 보이지만, 출발점 3개 기준으로 정리할 수 있음을 배웠다.

## BOJ 2211 — 네트워크 복구

- 이 문제는 MST가 아니라 **1번 정점을 루트로 하는 최단 경로 트리**를 만드는 문제임을 알게 되었다.
- 다익스트라에서 경로 전체(history)를 저장하지 않고, 각 노드의 `parent`만 기록하면 충분하다.
- 최단 거리가 갱신되는 순간의 이전 노드를 저장하면 복구해야 할 간선을 얻을 수 있다.

## 공통 개념 정리 (다익스트라 / 구현)

- `INF`는 실제 무한대가 아니라 “도달 불가능”을 표현하기 위한 센티널 값이다.
- `heapq`에서는 `(거리, 노드)` 형태로 넣어야 최소 거리 기준으로 pop된다.
- 파이썬에서 index 생성은 `range`, 기존 배열 순회는 `enumerate`가 정석이다.

## ⚠️ 오늘 공부하면서 실수하거나 헷갈렸던 부분

- `INF = 10e18`처럼 의미가 모호하고 타입이 불분명한 값을 사용하려 했던 점
- 다익스트라에서 history 전체를 heap에 담으려 했던 비효율적인 접근
- `$1`, `$2` 문자열 포맷이 언어 공통 문법이라고 착각했던 부분

# react와 react-dom이 나눠진 이유

react와 react-dom은 라이브러리가 분리되어있다

react는 아래와 같은 역할을 가지고있다
• 컴포넌트 모델
• JSX → React Element
• Hooks 규칙 (useState, useEffect 등)
• 렌더링 추상화 (Reconciliation)

React의 Reconciliation은 상태 변경 시 이전과 새로운 React Element Tree를 비교해 변경된 부분만 찾아내는 과정이며, 이를 통해 효율적인 렌더링이 가능해집니다.
